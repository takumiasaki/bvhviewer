# Architecture

## 目的

BVH ファイルの 3D 描画のためのアーキテクチャを検討する。

## アーキテクチャの概要

3D 描画には Qt Quick 3D を使用する。ただし、 `Skelton` は使用しない。
MVVM パターンに近い構成とし、データフローは「C++ Backend (Model/Manager) <-> QML (View/Animation Engine)」とする。

**基本方針**:
* **座標計算の委譲**: ローカル座標、グローバル（ワールド）座標の変換および描画は Qt Quick 3D に完全に委譲する。
* **データのフラット化**: C++内部ではBVHの木構造を保持してローカル座標・回転を計算するが、QML（View層）に対しては、階層を排除した「フラットな1次元リストモデル」としてデータを提供する。QML側での再帰的なノード構築や動的QML文字列生成は行わない。
* **アニメーション駆動**: QMLのアニメーションシステムが「再生すべきフレーム番号（整数）」を決定し、C++は指定されたフレームのデータを補間なしで即座にQMLへ供給する。

### 層の構成

1.  **QML View Layer & Animation Engine**
    *   `Scene`, `Camera`, BVHモデルの描画を担当。
    *   UI コンポーネント (`ListView`, `Slider` など) を担当し、C++ から提供されるデータを反映する。
    * C++から提供される「フラットなジョイントリスト」「フラットなボーンリスト」を `Repeater3D` 等で読み込み、球体（関節）や幾何学形状（ボーン）を空間上に配置・描画する。

2.  **Scene Manager (Controller & Adapter)**
    * `src/ui/bvhscehene` 以下。
    *   C++ クラス (`SceneManager`) として実装。QML と C++ Model の橋渡し役。
    *   **アニメーション制御**: 現在のフレーム番号に基づき、所持する各BVHオブジェクトにフレーム番号を設定する
    *   **状態管理**: モーション再生状態（再生中/停止、フレーム数）、照明、カメラ設定などを管理。
    *   **位置制御**: 各BVHオブジェクトの位置や回転を QML に提供するためのインタフェースを提供する。

3.  **BVH File Model 層**
    *   `src/core/bvhfile` 以下。
    *   BVHファイルのパース、データ構造 (`Channel`, `Frame`) の保持。
    *   ファイルI/Oの抽象化。

4.  **3D Data Model 層 (Internal State)**
    *   `src/ui/bvhdatamodel` 以下。
    *   SceneManager の内部で管理される「BVHデータのメモリ内表現」。
    *   QML へ各オブジェクトの Joint や Bone の座標データを提供する。
    *   **座標変換**: BVH File Model からモーションを適用後の各ノードのオブジェクト座標を計算する。ワールド座標は Qt Quick 3D に任せる。
    * 内部的にはBVH File Modelから読み取った木構造（Tree）として関節の依存関係を計算する。
    * 計算完了後、QMLが扱いやすい `QAbstractListModel`（フラットな配列構造）へと変換して公開する。

## 詳細設計方針

### 3D Data Model と BVH File Model の関係
*   1つのBVHファイルインスタンスに対して、1つの3D Tree構造（Jointリスト）が生成される。
*   3D Data Model は、BVH File Model から構造情報（Jointの親子関係、オフセット）を読み取り、アニメーションフレームごとにローカル座標を更新する。

### アニメーション駆動の流れ (Critical Path)
1.  ユーザー操作によりフレーム数が更新されると、`SceneManager` がそれを検知する。
2.  `SceneManager` は現在のフレーム番号に対応する各BVHオブジェクトのフレーム番号を設定する。
3.  `Bvh3DModel` は、現在のフレームの回転データを元に、各ノードのローカル座標を計算する。
4.  QML 側では、`SceneManager` から提供される座標データを使用して、Qt Quick 3D の 各ノードの座標値を更新する。

### ワールド座標系の扱い
*   通常描画においてワールド座標は Qt Quick 3D が内部で計算するため、C++側では算出しない。
*   **例外**: マウスヒットテストや外部エクスポートなどで「実際のワールド座標」が必要な場合のみ、`SceneManager` が QMLの各描画ノード（Modelなど） から最新の変換行列を取得する、またはC++側で木構造を走査してワールド行列を計算するなどの実装とする。

### UI との連携
*   構造木 (`TreeView`) は、「BVH File Model」から構築された静的な親子関係リストを使用する（アニメーションデータとは分離）。
*   表示・非表示の設定は BVH　オブジェクトの単位で管理し、QML から切り替え可能にする。

### QML へ提供するフラットなデータ構造
QML側での幾何学計算を一切排除するため、C++（3D Data Model）は以下の **2つの独立したフラットなリスト（QAbstractListModel）** を提供する。

1. **Joint List (関節点用のリスト)**
   * 主に球体（Sphere）を配置するために使用する。
   * 各要素は、C++側で計算済みの「オブジェクト座標」と「ジョイント名」を保持する。

2. **Bone List (骨・肉用のリスト)**
   * 主にシリンダー（Cylinder）などの幾何学形状を配置するために使用する。
   * 各要素は、2つのジョイントを繋ぐための以下のデータをC++側で計算済みとして保持する。
     * **position**: ボーンの中心点（または始点）の座標
     * **rotation**: 親から子へ向かう方向を表すクォータニオン（回転）
     * **length**: 親から子までの距離（シリンダーの Y 軸方向の `scale` にバインドする）

### UI との連携（修正版）
* QML側は、これら2つのリストをそれぞれ `Repeater3D` で読み込み、C++から渡された `position`, `rotation`, `scale` をそのまま適用して描画する。
* QML側での計算は基本的にオブジェクト座標系からワールド座標系への変換のみとする。(ただし、通常はQt Quick 3Dが自動的に処理するため、C++側でワールド座標を算出する必要はない)
