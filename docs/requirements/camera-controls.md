## 2. 目的とスコープ

本書は、BVH Viewer における **3D ビューのカメラ操作**（手動オービット、再生中の自動追従、明示フレーム）の要件と振る舞いを定義する。

### 2.1. 対象

- `Bvh3DView.qml` のカメラ・`OrbitCameraController`
- `SceneManager` / `CameraSettings` が公開するカメラ関連プロパティ
- サイドバー Settings タブのカメラ UI
- キーボードショートカット（`F`：Frame now）

### 2.2. 対象外

- マルチビューポート（将来拡張 F07）
- 照明のカメラ追従（将来拡張、§10 参照）
- 再生中の毎フレーム完全リフレーム（将来拡張、§10 参照）

### 2.3. 前提

- カメラは **オービットカメラ** モデル（回転中心 `cameraOrigin` + カメラ距離）を採用する（現行 `Bvh3DView.qml` と同様）。
- 手動操作（オービット・パン・ズーム）は要件 F05 として既に実装済み。
- スケルトン **選択** は `SceneManager.activeIndex`（`-1` で未選択）で表す。

---

## 3. 用語

| 用語 | 定義 |
| :--- | :--- |
| **注視点** | オービット回転の中心（`cameraOrigin.position`）。 |
| **フレーム** | 注視点の移動とカメラ距離の調整により、対象がビューポートに収まること。 |
| **Manual** | カメラの自動移動を行わないモード。ユーザー操作のみ。 |
| **Auto** | 再生中のみカメラを自動追従するモード。停止中はカメラを動かさない。 |
| **Frame now** | 現在の対象に対し、1 回限りのフレームを実行する明示操作。モードに依存しない。 |
| **Reset view** | カメラをアプリ既定の初期視点に戻す。 |
| **フレーミング対象** | 注視点・距離計算に用いるスケルトン集合（§5.1）。 |
| **playing** | シーンタイムラインが再生進行中かどうか（`SceneManager.playing`）。 |

## 4. カメラモード

### 4.1. Manual

- 再生中・停止中を問わず、**自動移動は一切行わない**。
- オービット・パン・ズームは常に有効。

### 4.2. Auto（停止中）

- カメラは **動かさない**（現在の視点を維持）。
- 選択変更・読込・オフセット変更等でも **自動フレームしない**。
- 次の再生開始時に追従を開始する **待機状態** とする。

### 4.3. Auto（再生中）

- **ここでのみ** カメラを自動更新する。
- **Play 開始時** に対象へ 1 回フレーム（§6.2）してから追従を開始する。
- 追従中は **注視点のみ** を連続更新し、**距離は Play 開始時に決定した値を維持**する（§6.3）。
- オービット操作は **無効**（自動追従と競合しないため）。

### 4.4. モード切替

- Manual / Auto は **いつでも** UI または API で切り替え可能。
- 再生中に Manual へ切替 → 追従即停止、オービット操作が有効になる。
- 再生中に Auto へ切替 → その時点で 1 回フレーム後、追従を開始する。

## 5. フレーミング対象

### 5.1. 対象スケルトンの決定

| `activeIndex` | フレーミング対象 |
| :--- | :--- |
| `-1`（未選択） | **表示中**（`visible == true`）の全スケルトン |
| `>= 0`（選択あり） | 選択スケルトンが **表示中** の場合のみその 1 体 |
| 表示中 0 体 | フレーミング対象なし（操作は no-op） |

非表示の選択スケルトンはフレーミング対象に **含めない**。

### 5.2. バウンディングボックス（AABB）

対象スケルトンごとに、**現在ポーズ**の全ジョイント位置から AABB を算出する。

```
各ジョイントの半径 = isEndSite ? endSiteDiameter/2 : jointDiameter/2
AABB.min/max      = ジョイント位置 ± 半径（各軸）
```

複数スケルトンの場合は、シーンオフセット（`sceneOffset`）適用後の AABB を **union** する。

**マージン:** 包囲 AABB を中心から **18%** 拡張する（`kDefaultFramingMargin = 1.18`）。

ジョイント直径は `Bvh3DView` の描画定数（`jointDiameter` / `endSiteDiameter`）と一致させる。

### 5.3. 注視点・距離の算出

| 項目 | 算出 |
| :--- | :--- |
| **注視点** | union AABB の中心 |
| **距離** | 各軸の半幅と FOV から必要距離を算出し、垂直・水平・奥行きの **最大値** を採用。カメラ俯仰角（既定 −20°）で `1 / cos(pitch)` 補正する |

### 5.4. 再生中の距離更新

- Play 開始時の 1 回フレームで距離を初期化する。
- 再生追従中は、ポーズ拡大により必要距離が増えた場合に **距離のみ即時に引き上げる**（縮小はしない）。
- 注視点がデッドゾーン内でも、必要距離の増加があればズームアウトする。

## 6. カメラ更新のトリガー

### 6.1. 自動更新（Auto + 再生中のみ）

| イベント | 動作 |
| :--- | :--- |
| Play 開始 | 1 回フレーム（§6.2）→ 距離を固定して追従開始 |
| ポーズ更新（`poseUpdated`） | 注視点のみ追視点へ移動（距離固定） |
| Pause | カメラ固定（追従停止、Auto のまま） |
| Play 再開 | 追従再開（新たな 1 回フレームは行わない） |
| Stop（先頭へ） | カメラは **動かさない** |
| 再生中の選択変更 | 1 回フレーム（短いイージング）→ 距離を再固定して追従継続 |
| 再生中の表示切替 | 対象が 0 体になれば追従停止。対象復帰時は 1 回フレーム |

### 6.2. 1 回フレーム（アニメーション）

- **所要時間:** 300 ms
- **イージング:** `Easing.OutCubic`
- **補間対象:** 注視点（3 軸）、カメラ距離
- **Shift + F**（Frame now）: 瞬時（0 ms）

### 6.3. 再生中追従

再生中の注視点追従では、ジョイントの微小な動きによるカメラの揺れを抑えるため、次の **安定化** を適用する。

| 手法 | パラメータ | 説明 |
| :--- | :--- | :--- |
| **デッドゾーン** | 包囲球半径 × 6%、最小 10 単位 | 現在の注視点からの目標中心の変位がこの範囲内なら更新しない |
| **指数移動平均（EMA）** | 基本 α = 0.15、最大 0.4 | デッドゾーンを超えた変位に対し、1 更新あたり目標へ α だけ補間する。変位が大きいほど α を増やす |

- Frame now・Play 開始時の 1 回フレームには安定化を **適用しない**（明示／初期フレームは正確に合わせる）。
- カメラ距離は Play 開始時に初期化し、再生中は §5.4 に従い **必要に応じて拡大のみ** 更新する。

### 6.4. Pause 中の Frame now

- Pause 後に `F`（Frame now）でカメラが動くのは **仕様どおり** とする。
- Frame now は「現在のポーズで対象をビューに収める」明示操作であり、停止中・一時停止中を問わず距離を含めて再フレームする。
- 再生追従で注視点がわずかにずれていた場合や、一時停止後のポーズが Play 開始時と異なる場合に、ユーザーが視界を整える手段となる。

## 7. 明示操作（モード非依存）

### 7.1. Frame now

- UI ボタンまたは `F` キーで実行。
- 現在のフレーミング対象に対し §6.2 の 1 回フレームを行う。
- Manual / Auto いずれのモードでも利用可能。
- 停止中の「全体を収めたい」用途を担う。

### 7.2. Reset view

- カメラを既定値に戻す。
  - 注視点: `(0, 200, 0)`
  - 俯仰角: `-20°`（Yaw `0°`）
  - 距離: `1300`
  - FOV: `45°`
- モードは変更しない。

## 11. 受け入れ条件

- [ ] Manual モードで再生中もカメラが自動移動しない
- [ ] Auto モードで停止中はカメラが自動移動しない
- [ ] Auto + Play 開始で対象がフレームされ、再生中は注視点が追従する
- [ ] Pause / Stop で §6.1 のとおりカメラが停止する（Stop では視点維持）
- [ ] 未選択時は表示中全スケルトン、選択時は表示中の選択スケルトンのみが対象
- [ ] 表示中 0 体で Frame now / 自動追従が no-op
- [ ] Frame now（`F`）が Manual / Auto 両方で動作する
- [ ] Reset view で既定視点に戻る
- [ ] Auto + 再生中はオービット操作が無効、Manual では有効
