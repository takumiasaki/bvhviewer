## 2. プロジェクト概要

本プロジェクトは、モーションキャプチャの標準的なフォーマットである **BVH (Biovision Hierarchy) ファイル**を複数読み込み、3D 空間上で視覚的に確認ができるデスクトップ向けのビューアアプリケーションを開発するものである。



---



## 3. システム構成・開発環境（制約事項）

アプリケーションの基盤として以下の技術スタックを採用する。



* **開発フレームワーク:** Qt 6.8 以降

* **UI テクノロジー:** Qt Quick (QML) / Qt Quick 3D

* **プログラミング言語:** C++（バックエンド・パース処理）, QML/JavaScript（UI・ビュー制御）

* **ターゲットOS:** Windows 10/11, Linux, macOS (主要なデスクトップ環境)

## 4. 機能要件



各項目末尾のタグは [実装状況](./backlog.md) に従う。



### 4.1. ファイル管理機能

* **F01: BVH ファイルのインポート** `[実装済]`

    * ローカルの BVH ファイル（`.bvh`）を **同一シーン内** に1体以上追加読み込みできること。複数ファイルは、File > Open・読込ボタン等による **追加読み込み** でシーンに載せる（1回のダイアログ操作で複数ファイルを選ぶ必要はない）。

    * 指定方法: アプリケーションのコマンドライン引数、メニューの File > Open、サイドバーの読込用ボタン

* **F02: 読み込み済みファイル一覧表示** `[実装済]`

    * インポートされたモーションファイルの名前や状態をリスト形式で確認できること。

    * 不要になったファイルをリストから削除（メモリ解放）できること。



### 4.2. 3D ビューア機能

* **F03: スケルトン（骨格）の3D描画** `[実装済]`

    * BVH の `HIERARCHY` 定義に基づき、関節（Joint）と骨（Bone）を3D空間に描画すること。

    * 複数ファイルを読み込んだ際、それぞれのスケルトンが識別できるよう、**読込順にパレット色を循環割当**して色分けすること（詳細は [スケルトン色選択仕様](./skeleton-colors.md)）。

    * 個別にスケルトンの表示・非表示を変更できること。

    * UI による操作はスケルトン単位とし、各スケルトン内の個別ジョイント／ボーンの表示切替は不要。

    * **F03-2: スケルトン色・配置の UI 編集** `[実装済]`

        * Active Scene パネル（Offset / Appearance / Remove タブ）で編集する。

        * Offset・色は **即時反映**。Reset to default のみで各初期値へ戻す。

        * 各スケルトンのシーン内配置（オフセット）を SpinBox による数値編集および Reset to default で調整できること。

        * 詳細は [スケルトン色選択・Active Scene 仕様](./skeleton-colors.md) を参照。

* **F04: 複数読込時のシーン構築** `[実装済]`

    * 複数ファイル読込時のシーン構築方法では重ならないように併置(自動的にオフセットを付与)する。

* **F05: カメラ操作（視点変更）** `[実装済]`

    * マウスまたはトラックパッド操作により、3D空間を自由に注視点変更（オービット）、平行移動（パン）、拡大縮小（ズーム）できること。

* **F05-2: 自動カメラ（再生中追従）** `[実装済]`

    * Manual / Auto モードを切り替え可能とすること。Auto では **再生中のみ** カメラが自動追従すること。

    * 未選択時は表示中の全スケルトン、選択時は表示中の選択スケルトンをフレーミング対象とすること。

    * Frame now・Reset view の明示操作を提供すること。

    * 詳細は [カメラコントロール要件・仕様書](./camera-controls.md) を参照。

* **F06: グリッド・床面の表示** `[実装済]`

    * 空間のサイズ感や位置関係を把握しやすくするため、地面（グリッド）を表示できること。



### 4.3. モーション再生コントロール機能



詳細な振る舞い・UI 仕様は [アニメーションコントロール要件・仕様書](./animation-controls.md) を参照。



* **F08: アニメーション再生・一時停止・停止** `[実装済]`

    * 読み込んだモーションを再生、一時停止、およびシークバーの先頭に戻すことができること。

    * 再生完了時は自動停止し、Play ボタン 1 回で先頭から再再生できること。

* **F09: タイムライン・シークバー操作** `[実装済]`

    * 読み込み済み全スケルトンに基づくシーン全体のタイムライン（シークバー）を表示し、任意の位置へジャンプできること。選択・表示状態でタイムライン長が変わらないこと。

    * 基準 `Frame Time` は最小値、総時間は各 `(frameCount - 1) × Frame Time` の最大値（BVH 慣習）とする。

    * ポーズ更新は表示中スケルトンのみ（非表示は描画・評価を省略）。

    * デフォルトは時間（秒）表示とし、フレーム表示への切り替えを可能とすること。

    * 停止中・再生中いずれのシーク後も、その位置から再生を継続できること。

    * 再生中は数値桁の変化でコントロール位置がずれないこと（スケルトン追加・削除によるレイアウト変更は可）。

* **F11: 複数モーションの同期再生** `[実装済]`

    * 複数のスケルトンを一斉に同期して再生する。

* **F11-2: 再生時間の同期と補間** `[実装済]`

    * BVHファイルに定義された `Frame Time` に基づき正確な速度で再生すること。

    * 描画レート（60FPS）とデータのフレームレートが異なる場合、必要に応じてキーフレーム間の線形補間（補間再生）を行うこと。



### 4.4. UI仕様

* **F12: UI基本構成** `[実装済]`

    * UI はメニューバーやステータスバーを持つ、デスクトップアプリケーション的な構成とする。

    * メニューからファイルのオープンやアプリの終了が可能なこと。

    * アプリケーションはサイドバー、3D ビュー、アニメーションコントロールから構成される。

    * サイドバーにスケルトンのリストやカメラコントロールを配置する。

    * アニメーションコントロールは 3D ビューの下部に配置する。

* **F13: UI スタイリング** `[実装済]`

    * Qt Quick Controls は **Fusion スタイル**を使用する。

    * 2D UI の色は **Palette** が定義する色を基本とし、直接の色指定（16 進リテラル等）は避ける。

    * 例外として、警告・エラー等の意味付きの状態色、および 3D ビュー（Qt Quick 3D）内の描画色は直接指定を許容する。

    * 詳細は [UI スタイリング方針](../design/ui-styling.md) を参照。

## 5. 非機能要件



### 5.1. 性能・パフォーマンス

* **描画の滑らかさ** `[未検証]` … 一般的なデスクトップ環境において、複数（例：3〜5体程度）のスケルトンを同時に描画した際、60 FPS 以上の安定したフレームレートでアニメーションが駆動すること。

* **ロード時間** `[未検証]` … 数千フレーム程度の標準的な BVH ファイルであれば、数秒以内にパースが完了し描画が開始されること。



### 5.2. ユーザビリティ (UI/UX)

* **UI の直感性** `[実装済]` … 直感的でモダンな UI を提供すること。Qt Quick Controls は Fusion スタイルを用い、OS のライト／ダーク設定に追従する（[UI スタイリング方針](../design/ui-styling.md)）。

* **3D 操作ヘルプ** `[未]` … 3D 空間での操作方法（ヘルプ）がアプリケーション内で確認できること。



### 5.3. 拡張性・保守性

* **レイヤー分離** `[実装済]` … パース処理（C++）と描画処理（QML）のレイヤーを明確に分離したアーキテクチャ（MVVMパターンなど）を採用すること。

* **フレーム駆動設計** `[実装済]` … C++側は純粋なデータモデルとパースに徹し、QML側のタイマーから指定されたフレームインデックス（整数値）に対して、即座にフラット化されたジョイントデータを返す設計とする。

* **データ構造の拡張性** `[実装済]` … 将来的な幾何学形状による肉付けに対応するため、C++からQMLへ渡すフラットなデータ構造には、各ジョイントの位置・回転だけでなく、親ジョイントとの位置関係（ベクトル等）を含められる拡張性を残す。

## 6. 将来の拡張機能



指示があるまでは検討・実装等の対象外となる機能。残件の管理は [backlog.md](./backlog.md) を参照。



* **肉付け（スケルトン形状）** `[将来]`

    * スケルトンに立方体、円柱、楕円体などで肉付けが可能なこと。

* **F04-2: シーン構築モードの選択** `[将来]`

    * 複数ファイル読込時のシーン構築方法について、初期位置（オフセット無し）、併置（自動的にオフセットを付与）などのモードを事前に指定できること。

* **F05-2: 自動カメラ（再生中追従）** — 実装済。§4 へ移動。[camera-controls.md](./camera-controls.md) 参照。

* **F07: マルチビューポート対応** `[将来]`

    * ビューアーはシーンに対して、複数のビュー（ビューポート）を構成できること。

    * 1面表示（俯瞰のみ）、3面表示（X/Y/Z軸）、4面表示（俯瞰 + X/Y/Z軸）を容易に切り替えられること。

    * 各ビューポートは、以下のいずれかのカメラ特性を持つ。

        * **俯瞰（透視投影）:** カメラコントロールやマウス操作で自由な視点変更（オービット等）が可能。1面表示および4面表示のメイン画面で使用する。

        * **X軸 / Y軸 / Z軸（平行投影）:** それぞれ対応する軸の視点（例：正面、真上、真横）にカメラを固定し、パースのない平行投影（正投影）で描画する。3面表示および4面表示のサブ画面で使用する。

    * 各ビューポート内のカメラ（視点・ズーム等）は、基本的に独立して操作可能であること。

* **F10: 再生速度変更（スロー再生）** `[将来]`

    * 等倍再生に加え、スロー再生（例：0.25倍、0.5倍）や倍速再生に対応すること。
